디자인시스템 v2 구축

"신뢰는 구조로, 친근함은 표면으로" — 같은 팔레트에 화면별 색 사용권을 두는 규칙 위에, 토큰·컴포넌트·기준화면·에이전트 규칙을 한 벌로 만들었습니다.

퍼플( #8C52FF ) 액센트를 폐기하고 코랄 #FF6B5C (모임 온기)와 크림슨 #F0384B (위험)을 분리했습니다. 핵심은 색 출입 권한 규칙 : money 화면 (정산·금액)은 블루·잉크·민트만, community 화면 (모임·온보딩)은 코랄 액센트 허용. 기준화면은 meeting-detail.html 입니다.

클로드코드 개발 시 활용법

새 화면을 만들 때 이 순서를 따르면 meeting-detail과 자동으로 일관됩니다.
AGENTS.md를 참조하게 하면 아래 내용이 적용됩니다

template.html 복사로 시작

폰 프레임·헤더·CSS 링크가 들어간 빈 골격에서 출발. tokens.css + components.css 를 항상 둘 다 링크.

화면 종류를 선언

루트 .screen 에 surface-money (정산·금액) 또는 surface-community (모임)를 건다 → 색 권한이 자동 적용.

컴포넌트는 클래스 조합

CSS를 새로 짜지 말고 .btn .card .badge .stepper .slab .field 등을 쓴다. 예시는 components.html 참조.

토큰만 사용 · 이모지 금지

색·간격은 var(--…) 만(새 hex 발명 금지). 금액엔 class="num" . 글리프는 icons.svg .

AGENTS.md 체크리스트로 마감

완료 전 6단계 체크(CSS 링크·surface·블루 CTA·num·이모지 0·토큰만)를 통과시킨다.

절대 규칙 4가지

색 출입 권한 money 화면에 코랄·크림슨 금지. 돈 CTA는 항상 블루.
토큰만 새 색·간격·라운드 발명 금지. tokens.css 변수만.
이모지 금지 모든 글리프는 icons.svg 인라인 SVG.
마스코트는 모먼트만 온보딩·빈상태·완료 축하에만. 정산 화면 금지.
